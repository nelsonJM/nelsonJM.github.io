Creating Components in Figma: My Takeways

I used the tips from Thomas Lowry's article to decompose the City of Austin's Office of Design & Delivery homepage into components using Figma.

Get the Figma file for an in-depth look.

Download the Figma File

Background

I've found myself in the following situation: a designed webpage needs to be broken down into useful measurements that developers can use to guide build efforts while simultaneously remaining accessible to designers for additional changes.

I've also encountered these associated obstacles:

  1. focusing on semantics too soon, e.g. "h1s should look like this..."
  2. premature "componentizing"

For this exercise, I assumed that a designer has presented me with a high-fidelity concept of the following homepage:

Overcoming Obstacle 1: Text Styles

Trying to establish the global styles for all the elements: h1, h2, h3, anchors, paragraphs, etc. in a design tool vs in code. Figma's color and text styles are great because they allow designers to focus on how elements should be desired while still maintaining a constraint on the number and kind of styles.

For example, I identified the text styles in this homepage and Figma conveniently lists them on the design tab:

A developer can take this list of styles and easily put it into a scale with CSS classes. The designer doesn't need to worry about whether they should use the H1 vs the H2.

Overcoming Obstacle 2: Scales vs "Another component"

Another tendency is to create a component when it isn't necessary. For example, instead of creating a component for each section on the homepage, I used utility-class thinking to make the spacing and background colors reusable. I created a small scale for the spacing above each heading and below each paragraph. The background colors (white and grey) are specified as 'bg-white' or 'bg-grey-light. As such, they can be applied to backgrounds that may be used in non-section parts of the UI.

Conclusion

These are a couple of ways that Figma and Utility-class thinking is helping me solve the problem of communicating design requirements to developers while preserving the ability to make design changes in the design tool.
