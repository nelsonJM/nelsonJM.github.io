I did not design this modal. I'm posting it here because I learned a few things while implementing it using the Bootstrap framework.

Breaking Up the Design

I began by looking at the screenshot above and identifying independent areas. The process looked like this: On one side of my screen I had the mockup and on the other I had an empty nvALT document. I looked at the mockup and said, "There's the notion of 'this call that's happending right now'", then I would enter ".call" in the nvALT doc. ".call" is the classname I will give to the 'call' component in the HTML. "This is a contact, so I'll write a classname called '.contact'.". Soon I had a list of classnames representing each independent area of the design. Later, I thought about the data as an outline of objects. Something like this:

  • Call
    • History
    • Duration
    • Registers
  • Contact
    • Name
    • Bio
    • Phone Number
    • ...

This helped me identify components (like 'Call') and sub-components (like 'Contact') and make distinctions such as "call-history" and "contact-history" (different sub-components).

Naming Component and Sub-components (How I Write CSS)

I've been trying Mark Otto's approach to writing CSS as outlined in his talk, mdo-ular CSS. The result is CSS that looks something like this:

  • component
  • component-modifier
  • component-subcomponent
  • component-subcomponent-modifier

Watch the video for a great explanation and examples.

Coding

I downloaded a copy of Bootstrap and began coding using my list of classnames as a starting point for each component. Initially, I coded most of the components from scratch and used Bootstrap mainly for the grid system, JavaScript plugins and helper classes like 'pull-right'. I soon ran into limitations relying on the grid system for layout and took a step back to re-evaluate the design. I began to notice the patterns within patterns, which led me to dig around the Bootstrap documentation for solutions.

Leaning on Bootstrap

Pouring over the Bootstrap documentation brought my attention to areas of the framework I hadn't noticed before. Helper classes like ".text-right" were familiar, but ".text-muted" and ".center-block" helped me position elements quickly and with less code. By far the most utilized pattern in this design is the "media object" and its variations. These are very specific uses of Bootstrap, but I did learn two broader, slightly higher level lessons:

  • Bootstrap encouraged me to refine my understanding of what constitutes a component. One component could house a few smaller components.
  • Write CSS in a way that let's you write flexible HTML.

Where to From Here?

Of course I have plenty to learn. My hope is that this recap of the build process will spark new ideas and connections in my brain. It is an exciting and humbling experience.
